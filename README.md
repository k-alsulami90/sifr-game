# صِفر — Sifr Game

لعبة معلومات جماعية بأسلوب **Pointless** (منتج أصلي، بلا أي علامات منسوخة)، عربية
بالكامل بواجهة **RTL**. تُلعب في غرفة مع الأصدقاء: شخصٌ يفتح **لوحة التحكّم**
(`/admin`) على جواله، وآخر يفتح **شاشة العرض** (`/display`) ويعكسها على التلفاز.

> الفكرة: **أندرُ إجابة… أعلى فوز**. كل إجابة لها درجة شيوع من ١ إلى ١٠٠ (الأشهر =
> أعلى رقم = أسوأ). الإجابة النادرة جدًّا غير المذكورة = **صِفر** وهي أفضل نتيجة.
> الفريق صاحب **أعلى مجموع** في الجولة يُقصى، وآخر فريق صامد هو البطل.

تم بناء الواجهة لتطابق تصميم **Sifr Game** من Claude Design.

---

## التقنيات

- **React + Vite**
- **Firebase Realtime Database** للمزامنة اللحظية بين الجهازين
- **Tailwind CSS**
- **Framer Motion** للانتقالات
- **Web Audio API** للأصوات (تُولَّد برمجيًّا — لا حاجة لملفّات صوت)
- عربية بالكامل، `dir="rtl"`

## التشغيل محليًّا

```bash
npm install
npm run dev
```

ثم افتح:

- لوحة التحكّم (المُضيف): `http://localhost:5173/admin`
- شاشة العرض (التلفاز): `http://localhost:5173/display`

يعمل الموقع بدون Firebase لتجربة لوحة التحكّم وحدها، لكن **المزامنة بين الجهازين
تتطلّب إعداد Firebase** (خطوة تالية).

## إعداد Firebase

1. أنشئ مشروعًا في [Firebase Console](https://console.firebase.google.com/).
2. من **Build → Realtime Database** أنشئ قاعدة بيانات (اختر أقرب منطقة).
3. قواعد الأمان للتجربة (تسمح بالقراءة والكتابة — استخدمها للّعب الخاص فقط):

   ```json
   {
     "rules": {
       "rooms": {
         ".read": true,
         ".write": true
       }
     }
   }
   ```

   > لإنتاج عام، قيّد الكتابة (مثلًا عبر مصادقة مجهولة أو مفتاح للغرفة). كل
   > الكتابة في هذا التطبيق تتمّ من شاشة المُضيف فقط، وشاشة العرض للقراءة فقط.

4. من **Project settings → Your apps → Web app** انسخ قيم الإعداد.
5. انسخ `.env.example` إلى `.env.local` واملأ القيم:

   ```bash
   cp .env.example .env.local
   ```

   ```
   VITE_FIREBASE_API_KEY=...
   VITE_FIREBASE_AUTH_DOMAIN=...
   VITE_FIREBASE_DATABASE_URL=https://<project>-default-rtdb.firebaseio.com
   VITE_FIREBASE_PROJECT_ID=...
   VITE_FIREBASE_STORAGE_BUCKET=...
   VITE_FIREBASE_MESSAGING_SENDER_ID=...
   VITE_FIREBASE_APP_ID=...
   ```

6. أعد تشغيل `npm run dev`.

## كيف تُلعب

1. على جوال المُضيف افتح `/admin` — يُنشأ تلقائيًّا **رمز غرفة من ٤ أرقام** ويظهر
   أعلى الشاشة.
2. على جوال آخر (موصول بالتلفاز) افتح `/display`، أدخل الرمز، واضغط **انضمام**.
   زر **⛶** يفعّل ملء الشاشة، وزر **🔊** للصوت.
3. في لوحة التحكّم: اضبط عدد الفرق وأسماءها، وليختر كل فريق **فئة مضمونة**، ثم
   **بدء اللعبة**.
4. لكل دور: تظهر الفئة على الشاشة (سحب يشبه ماكينة الحظ)، ثم السؤال. أدخل إجابة
   كل فريق (بحث تلقائي في بنك الإجابات) واضغط **كشف النتيجة** — يبدأ العدّ التنازلي
   على لوح المئة نقطة. أو استخدم زرّي **صفر!** أو **خطأ / انتهى الوقت**.
5. بعد دورتين تُجمَع نقاط الجولة، ويُقصى الأعلى. عند التعادل يحسمه المُضيف.

### أزرار العرض الثلاثة للنتيجة

- **إجابة عادية (١–١٠٠):** يتوقّف العدّ عند الرقم.
- **صِفر! (إجابة نادرة صحيحة = ٠):** انفجار ذهبي + قصاصات + نغمة فوز.
- **خطأ / انتهى الوقت (= ٠ لكن خاطئة):** استيلاء أحمر + علامة ✕ + نغمة هبوط.

> الصورتان للصفر (الصحيح مقابل الخاطئ) مُتتبّعتان بشكل منفصل في الحالة
> (`outcome: "pointless"` مقابل `"wrong"`) ومعالجتان بصريًّا بلا أي لبس.

## بنك الأسئلة

الملف: [`public/questions.json`](public/questions.json). كل سؤال:

```json
{
  "id": "q001",
  "category": "عواصم دول أفريقية",
  "question": "اذكر عاصمة دولة أفريقية",
  "bounded": true,
  "answers": [
    { "text": "القاهرة", "score": 97, "note": "مصر — الأشهر على الإطلاق" },
    { "text": "ليلونغوي", "score": 1, "note": "مالاوي — أندر إجابة" }
  ]
}
```

- `score` من ١ إلى ١٠٠ (شيوع الإجابة؛ الأعلى = الأشهر = الأسوأ).
- `score: 0` محجوز للإجابة الصحيحة النادرة غير المُدرجة (تُدخل عبر زر **صفر!**).
- الفئات تُشتقّ تلقائيًّا من حقل `category`. أضف أسئلة/فئات بتحرير هذا الملف فقط.

الفئات المبدئية: عواصم دول أفريقية، دول تبدأ بحرف الكاف، الحذاء الذهبي للدوري
الإنجليزي، أندية الدوري الإيطالي، بطولات التنس الكبرى، بالإضافة إلى جغرافيا وعلوم
وأبطال دوري أبطال أوروبا للتنويع.

## النشر على Vercel

1. ادفع المشروع إلى مستودع Git.
2. في Vercel: **New Project** واختر المستودع. سيتعرّف على Vite تلقائيًّا
   (`build` → `npm run build`، مجلد الإخراج `dist`).
3. أضف نفس متغيّرات البيئة (`VITE_FIREBASE_*`) في **Settings → Environment
   Variables**.
4. التوجيه: المشروع يستخدم توجيهًا من جهة العميل (`/admin`, `/display`). إن واجهت
   صفحة 404 عند التحديث المباشر لمسار فرعي، أضف ملف `vercel.json` في الجذر:

   ```json
   { "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
   ```

## بنية حالة Firebase

```
rooms/{roomCode}/
  status               "setup" | "playing" | "game_over"
  teams                [{ id, name, dot }]
  eliminated           [teamId]
  settings             { rounds, teamCount }
  sessionCategories    [أسماء الفئات لهذه الجلسة]
  currentRound         رقم
  currentPass          1 | 2
  turnOrder            [teamId]
  answeringPos         رقم
  answeringTeamId      teamId
  turnPhase            "spin" | "question" | "countdown" | "reveal" | "elimination"
  currentQuestion      { id, category, question, answers:[{text,score,note}] }
  usedAnswers          [{ text, score, teamId }]
  log                  [{ round, pass, teamId, score, outcome, answer }]
  lastReveal           { teamId, answerText, score, outcome, at }
  targetScore          رقم
  outcome              "scored" | "pointless" | "wrong"
  timer                رقم   timerRunning  منطقي
  eliminatedThisRound  teamId | null
  tieCandidates        [teamId]   (عند التعادل، يحسمه المُضيف)
  winner               teamId | null
  spinToken,
  countdownToken       عدّادات لإعادة تشغيل حركات العرض
  updatedAt            طابع زمني
```

كل الكتابة من **لوحة المُضيف**؛ شاشة العرض **للقراءة فقط** وتُشغّل الحركات
(السحب، العدّ التنازلي، الاحتفال) محليًّا اعتمادًا على هذه الحالة.

## ملاحظات حول المنطق

- المُضيف هو المحرّك المُعتمَد (`src/game/controller.js`): يدير الأطوار بمؤقّتات
  ويكتب اللقطة إلى Firebase؛ شاشة العرض تعكسها وتُحرّكها.
- توقيتات السحب والعدّ التنازلي مُشتقّة من دوالّ حتمية مشتركة
  (`src/game/timing.js`) كي تتزامن الشاشتان دون أن تكتب شاشة العرض أي شيء.
- **آلية الجولة:** كل جولة سؤالٌ واحد ودورتان عليه. تظهر الفئة (سحب) مرّة واحدة في
  بداية الجولة، ثم يجيب الجميع بالترتيب التصاعدي (الدور الأول)، ثم **بنفس السؤال**
  بالترتيب المعكوس (الدور الثاني). لا يُعاد استخدام أي إجابة طوال الجولة (الدورين).
  يُجمع لكل فريق ناتج دوريه، والأعلى مجموعًا يُقصى (يحسم المُضيف التعادل).
